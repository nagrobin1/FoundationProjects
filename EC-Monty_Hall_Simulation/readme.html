Introduction

In this project you will write C++ code that simulates playing a game a number of times. In this game the player is presented with three doors. One of the doors contains a prize and the other two do not.

The player is asked to first pick one of the three doors. What makes this game interesting is that a game host then opens one of the two doors the player did not pick, always a losing door. The player is then given the choice to stay with their original pick, or switch to the remaining unopened door.

What your code will do for each game, is tally whether the player wins by staying with their original choice, or by switching doors. At the end of the simulation we will know which strategy provided the best odds of winning.

In this project you will learn about:

  • Using the rand() function
  • Simulations

History

The game was originally described by Steve Selvin in a letter to American Statistician in 1975. And became famous as a reader question to Marilyn vos Savant at Parade magazine in 1990. It was quoted as follows:

Suppose you're on a game show, and you're given the choice of three doors: Behind one door is a car; behind the others, goats. You pick a door, say No. 1, and the host, who knows what's behind the doors, opens another door, say No. 3, which has a goat. He then says to you, "Do you want to pick door No. 2?" Is it to your advantage to switch your choice?

The game gets its name from a TV game show host, Monty Hall, who hosted the show Let's Make a Deal.

You can read tons more about it at Wikipedia, which includes various solutions.

Details

For this assignment, we assume we are playing a fair game, such that the each of the 3 doors has a 1/3 chance at being picked as the winning door.

You can use the rand() function to run your simulation and assume that it provides numbers from a uniform distribution. Don't forget to call srand(time(0)) to seed the random number generator.

(Note that rand() is fine for our class purpose, but in general should not be used in production code)

Testing:

There is no script this time. Since we are simulating a random event, the answer will not be the same every time. Instead, you will know you are correct if the strategy of switching is close to winning 66.7% of the time and staying is close to 33.3% of the time.

You will also have to check the output. The host should never open the winning door, or the door that the user has picked.
